# Security Policy

## Reporting Vulnerabilities
If you discover a security vulnerability in the Parallaxin codebase
or infrastructure, please report it to: parallaxin@proton.me

Do NOT open a public issue for security vulnerabilities.

## Scope
- Code vulnerabilities (XSS, injection, build pipeline compromise)
- Infrastructure vulnerabilities (hosting, DNS, deployment)
- Content integrity issues (unauthorized modification of published records)

## Out of Scope
- Factual disputes about content (use the standard issue process)
- Availability issues with external source URLs

## Response
We will acknowledge receipt within 72 hours and provide
an initial assessment within 7 days.

## No Bounty
Parallaxin is a zero-budget open source project.
We cannot offer financial rewards but will credit
reporters in the security advisory if desired.
